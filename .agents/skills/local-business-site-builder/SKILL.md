# Local Business Site Builder

## Workflow

1. Read `AGENTS.md`, the selected client brief, and the current files in `src/config`.
2. For a real business, verify every fact against public sources. Record sources in the brief. Never invent credentials, reviews, claims, pricing, or contact details.
3. Audit the existing web presence using `references/audit-rubric.md`.
4. Update only business-specific configuration and approved images first. Change components only when the client genuinely needs a different layout.
5. Keep fictional or unofficial concept notices prominent until approval.
6. Preserve the local-only safe demo form unless a real integration is explicitly requested.
7. Run the production build, then inspect desktop and mobile layouts. Test every CTA, anchor and form state.
8. Produce an audit using `audits/prospect-audit-template.md`.
9. Draft permission-based outreach using `outreach/templates.md`. Never send it without user approval.

## Completion Gate

- Verified facts are separated from assumptions.
- Mobile layout has no overlap or horizontal scrolling.
- Phone and WhatsApp links use valid formatting.
- Demo form sends and stores no data.
- Build succeeds.
- Concept status is clearly labelled.
