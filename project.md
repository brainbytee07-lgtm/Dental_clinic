# Project

## Working rules

- Keep the site simple, fast, accessible, and mobile-first.
- Store client-specific facts in `src/config`; do not hardcode them into layouts.
- Never invent claims, reviews, certifications, credentials, prices, or guarantees for a real business.
- Mark fictional and unofficial concept demos clearly.
- Preserve the safe demo form unless the user explicitly requests and approves a real integration.
- Avoid adding dependencies unless they solve a clear production need.
- Test the production build and responsive layout after material changes.
- Do not contact prospects or publish a personalised concept without user approval.

## Design workflow

- For any website, landing page, or marketing page, use the installed `design-taste-frontend` skill first.
- Use Impeccable for the project-level design workflow, especially when shaping, critiquing, polishing, or adapting a UI.
- Prefer the lightest workable prototype, then refine the layout, spacing, typography, and visual rhythm.
- Keep the first version honest and usable before making it fancy.

## Clinic demo strategy

- Lead with patient enquiries, not generic beauty.
- Use fake clinic names only for demos, not for real prospects.
- Include a before-and-after sample that shows the upgrade from weak or missing website to enquiry-focused clinic site.
- Keep the pitch simple enough for WhatsApp, PDF, or a quick walkthrough.

## Deployment

- Build command: `npm run build`
- Output directory: `dist`
- Vercel and Netlify both work for deployment previews.
